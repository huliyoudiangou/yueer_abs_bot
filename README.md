# 悦耳声阅 Bot

用户使用说明见根目录的 `abs_bot_README_user_manual.md`。

## 邀请码注册有效期

- 邀请码注册预校验读取数据库失败时会提示稍后重试，不会把系统故障误报为邀请码无效。
- 邀请码注册默认有效期由 `ACCOUNT_VALID_DAYS` 控制，当前默认 `30` 天。
- 如果同一 Telegram 用户曾经注册过并保留本地档案，再次使用邀请码注册时，账号有效期至少会补到本次注册默认有效期；若旧有效期更长，则保留更长的旧有效期，不叠加、不缩短。

## 开放注册与服务补偿

- 超级管理员可在私聊管理面板通过 `超级管理员控制台 → ⚙️ 系统配置 → 🚪 开放注册` 使用 `开放注册`，临时允许用户不输入邀请码注册；可选择限制本轮注册总人数，或限制开放分钟数（最长 7 天），同一时间只允许一个开放活动，也可提前关闭。`INVITE_REQUIRED=false` 仍表示全局免邀请码；管理面板开放注册是在 `INVITE_REQUIRED=true` 时提供的临时放行。
- 人数模式在 ABS 开户前事务预占名额，ABS 开户或本地档案写入失败会释放名额；成功注册后名额转为已完成，避免并发超卖。
- 超级管理员可在私聊管理面板通过 `超级管理员控制台 → ⚙️ 系统配置 → 🛡 用户补偿` 使用 `用户补偿`，选择统一发放积分、ABS 有效天数，或两者同时发放，并填写事故播报。任务会对创建时的本地用户快照逐人幂等处理，积分写入流水，完成后私聊用户并向发起管理员回执。
- ABS 天数补偿仅适用于已绑定 ABS 且具有有限到期日的账号；无限期账号保持无限期，未绑定 ABS 的本地积分档案只接收积分补偿。私聊失败不回滚已到账资产；ABS 恢复失败会在管理员回执中提示人工核查。
## 账号安全

- `修改密码` 需要先校验安全码，再同步 ABS 新密码；本地档案暂时读取失败时会提示稍后重试，不会继续校验空安全码或向空 ABS 账号同步密码。
- `账户安全`、换绑安全码校验、仅解绑和删号注销的安全码校验都会先读取本地档案；读取失败时会提示稍后重试，不会误当成未绑定或继续执行高危动作。
- `注册账户`、`绑定账号`、`听书报告` 和 `删号注销` 都会先确认本地档案状态；读取失败时会提示稍后重试，不会误当成未注册、未绑定或直接执行注销。`听书报告` 读取 ABS 主统计失败或解析失败时会提示稍后重试，不会误显示为暂无收听；累计实际听书优先使用 ABS 官方 `totalTime`，今日实际听书按北京时间当天 `days` 明细及已落库的当天统计展示，均保留两位小数；仅书籍完成/在听进度读取失败时，报告保留时长信息并把对应数量显示为“读取失败”。

## 积分兑换

- 邀请码和 30 天续期卡兑换会先读取当前价格配置；读取或解析失败时中止兑换，不会按默认价扣分。兑换成功后会私聊发送卡密；续期卡会提示是否二次确认后立即使用，确认后直接为当前正式账号延长有效期。兑换邀请码时，只有新人体验账号或尚未注册 ABS 账号才会提示是否立即使用：体验账号确认后转正，未注册账号确认后进入开户注册流程并跳过再次输入邀请码；已有正式账号不会出现直接使用提示。使用积分兑换的续期卡仅限当前持有人本人使用，或通过交易行出售变现积分；成交后续期卡归属转给买家。

## 播放异常风控

- 后台每日 03:00 后按北京时间完整自然日统计 ABS 原始播放时长，检测目标为“昨天”和“前天”，不使用天道压制后的净修为。
- 风控从 `listening_abuse_effective_start_day` 记录的北京时间日期起算；生效日前的历史收听时长既往不咎，不参与连续两日判断。
- 首次初始化风控生效日写入失败时，会跳过本轮巡检并通知超级管理员，不会拿未持久化的临时日期继续扫描。
- 后台读取风控生效日或末次扫描日失败时，会跳过本轮巡检并记录最近错误、通知超级管理员；状态不明确时不会冻结、恢复或重置生效日。
- 巡检读取某用户当天或前一天每日听书统计失败时，会记录诊断并跳过该用户本轮判断；不会把数据库异常当成 0 小时后继续误判。
- 每日净修为与播放异常风控都只使用 ABS 官方 `listening-stats.days` 的北京时间自然日值；用户历史听书会话列表不再参与跨日重分配或实时墙钟补偿，避免多条历史会话叠加造成当日误刷满。
- 后台风控已完成扫描但末次扫描日、扫描时间或最近错误状态写入失败时，会通知超级管理员人工核查，避免下一轮重复扫描同一天。
- 单日原始播放时长 `> 15` 小时会私聊提醒一次；连续两个自然日都 `> 15` 小时会临时暂停 ABS 听书权限 `24` 小时。
- 单日异常提醒会先在同一数据库事务内创建提醒记录并写入 `LISTENING_ABUSE_WARNING` 审计，事务提交后才私聊用户；若审计或记录写入失败，本轮不会发送未审计的提醒。
- 触发暂停时 Bot 会私聊用户，并在公告群提示账号因播放异常暂停一天；本次风控不扣积分、贡献、声望或修为。
- 暂停满 `24` 小时后后台自动恢复；若账号已到期、已被管理员封禁或绑定的 ABS ID 已变化，则不会自动恢复，避免误解封。
- 自动冻结或恢复失败会写入高危审计；若失败审计也无法写入，会通知超级管理员人工核查。
- `我的信息` 和管理员 `查询用户` 会同时展示本地档案状态、播放异常临停记录和 ABS 服务端 `isActive` 状态；若本地与 ABS 服务端不一致，会提示待核查，不再简单显示为正常。

## GitHub 福利

- 活动入口为 `活动 -> GitHub福利` 或私聊发送 `github福利`；群内发送只提示转私聊，不在群内校验或发码。
- 领取条件固定为 GitHub 账号注册满 `5` 年，并能把 Bot 生成的专属校验码放入该 GitHub 账号 Bio。
- 校验码按 TG 用户随机生成，有效期 `1` 小时；放入 Bio 后发送 `校验github福利 GitHub用户名` 完成校验，用户名也可写成 `@用户名` 或 `https://github.com/用户名` 链接，不接受 `http://`、带账号密码信息或非 `github.com` 主机的链接。
- 校验码过期后需要重新发送 `github福利` 获取新码；若过期状态暂时写入失败，Bot 会提示稍后重试，不会继续发放或误报正常过期。
- 奖励按当前 TG 账号状态发放：未绑定 ABS 账号发 `1` 个邀请码；已绑定 ABS 账号发 `150` 天续期卡。成功后仅通过私聊发送给领取人。
- 每个 TG 账号最多领取一次，每个 GitHub id 最多领取一次；名额由超级管理员通过后台命令设置。
- 可在 `.env` 配置 `GITHUB_API_TOKEN` 使用无权限 GitHub PAT 提升 API 限额；不配置时仍可匿名校验，但多人集中领取容易触发 GitHub 限流。
- 超级管理员可用 `查看github福利`、`设置github福利名额 数量`、`开启github福利`、`关闭github福利` 管理活动；超管面板 `活动运营` 也提供名额设置按钮。写入命令必须私聊、填写原因并二次确认，且写入审计。`查看github福利` 的开放状态、剩余名额、已领取和有效待校验计数若暂时读取失败，会明确显示读取失败，不会用关闭或 0 代替。

## 积分抽奖

- 定时抽奖创建支持分钟数，也支持 `YYYY-MM-DD HH:MM:SS` 或 `YYYY-MM-DD HH:MM` 固定时间。
- 固定开奖时间按北京时间解析。
- 创建抽奖的活动、奖品和审计会在同一事务内提交；强制开奖会在开奖事务内写审计；取消抽奖会在同一事务内关闭活动、全额退还参与积分、写退款流水和取消审计。
- `抽奖详情 活动ID` 读取活动、奖品、中奖记录或参与人数失败时会明确提示读取失败，不会把数据库异常误显示为活动不存在、空奖品、无中奖记录或 0 人。
- 领奖暗号错误次数过多会暂停受理；若安全锁状态暂时读取失败，系统会提示稍后再试，不会继续处理暗号。`人工 奖品名称 份数` 可创建联系管理员领奖的特殊奖品；`卡密 奖品名称|卡密内容` 可逐行导入非本服自定义卡密。两类奖品仍沿用开奖后 2 小时内私聊 Bot 发送暗号的领奖流程，自定义卡密仅加密保存并在成功领奖时私发。

## 世界 Boss

- 世界 Boss 击杀成功时生成 `30` 积分 / `10` 份 Boss 红包，并向天道奖池注入 `10` 积分。
- Boss 红包仅限本期已参加 Boss 的修士发送 `抢` 领取；普通积分红包和天道机缘红包不受该限制。
- 制作积分红包输入金额后，如果钱包暂时读取失败，Bot 会提示稍后重新输入金额，不会误判为余额不足；红包份数范围为 `3-100`，总积分仍不得小于红包份数。
- 红包领取成功后的余额或抢空榜读取失败时会明确显示读取失败，不会把数据库异常误显示为 0 积分或空榜。
- 世界 Boss 开启后最后 `15` 分钟停止新道友加入，已参加修士仍可继续造成伤害。
- `参加Boss` 需要当前有效且未暂停的 ABS 听书账号；已过期、管理员暂停或播放异常临停中的账号不能参加。
- 结算时若参与者本地账号已不存在或已解绑 ABS，将不计入最终伤害和奖励。
- 实时战榜刷新时也会清理已失效参与者的旧伤害展示，避免血量和排行沿用陈旧伤害。
- 世界 Boss 伤害按参与后 Boss 时段实际听书小时 × `30` ×（1 + 修为加成 + 宗门科技）计算，公告、状态、排行和结算消息会安全展示 Boss 名称，避免特殊字符破坏 Markdown。
- `参加Boss`、`Boss状态` 和 `Boss排行` 读取当前或最近 Boss 状态失败时会明确提示稍后重试，不会把数据库异常误显示为当前没有 Boss。
- `参加Boss` 成功后若参与记录重读失败，当前基线实际听书会显示读取失败，不会误显示为 0 小时；参加记录仍以已提交写入为准。
- `Boss排行`、实时战榜和结算公告读取排行失败时会明确提示读取失败，不会把数据库异常误显示为暂无参与者。
- 有效参与者个人奖励门槛仍为最终伤害 `>= 3`；击杀时有效参与者积分 `+1`、宗门贡献 `+3`，前三名和 Top 10 额外奖励保持原规则。

## 宗门秘境

- 宗门秘境分为普通、高阶和限时档位；普通秘境保持 `听潮秘境` 旧口径，高阶秘境默认要求宗门 3 级且参与者至少筑基，限时秘境默认关闭，供运营活动启用。
- 每个宗门每周最多开启 2 次秘境，普通、高阶和限时秘境合并计算，按北京时间周一 00:00 重置。
- 普通秘境开启消耗为 `100 + 宗门等级 * 30` 宗门资金；高阶秘境继续使用独立高阶成本。
- 秘境奖励按开启时保存的档位规则快照结算，管理员后续改配置不影响进行中的秘境；若活动配置快照缺失或损坏，进入和结算会中止并等待修复，不会回退到当前配置继续发奖。
- 结算时若参与者本地账号已不存在或已解绑 ABS，将不计入最终护道者、奖励和掉落。
- 实时榜刷新时也会清理已失效参与者的旧奖励展示，避免榜单显示陈旧积分、贡献、声望或掉落。
- 秘境奖励按 ABS 原始听书增量结算，并按加入时间到秘境结束的墙钟时长封顶；默认最低有效秘境净修为为普通/限时 `0.2` 小时、高阶 `0.75` 小时。
- 积分按普通/限时 `floor(小时*6)`、高阶 `floor(小时*7)` 后叠加固定境界加成，普通/高阶/限时默认上限分别为 18/30/20。
- 贡献和声望仍按档位倍率、护道加成和封顶规则结算；档位可配置全额计入时长、后续折算比例、贡献/声望倍率、奖励上限、护道加成和掉落池。
- `宗门秘境明细 [用户ID]` 可查看原始听书、墙钟封顶、压制后净修为、护道加成、最终奖励和掉落；查看他人明细仅限宗主、长老或超级管理员。
- 超级管理员可用 `查看秘境配置`、`设置秘境档位`、`设置秘境倍率`、`设置秘境掉落` 维护秘境规则；`设置秘境倍率` 的积分百分比字段仅兼容保留，当前积分按固定公式结算。写入必须私聊、填写原因并二次确认；配置读取失败时不会按默认配置继续开启或覆盖写入。

- 宗门秘境开启后会发送一条常驻实时榜，后台约每 2 分钟编辑同一条消息刷新；发送 `宗门秘境排行` 会立即按当前 ABS 原始听书增量手动刷新榜单。
- 秘境结束后再次发送 `宗门秘境` 会提示当前没有可参加的秘境；最近一次榜单仍可通过 `宗门秘境排行` 查看。
- `宗门秘境`、`进入宗门秘境`、`宗门秘境排行` 和 `宗门秘境明细` 读取当前或最近秘境状态失败时会明确提示稍后重试，不会把数据库异常误显示为没有秘境记录。
- `进入宗门秘境` 成功后若参与记录重读失败，当前境界和基线净修为会显示读取失败，不会误显示为凡人或 0 小时；进入记录仍以已提交写入为准。
- `宗门秘境排行`、实时榜和结算公告读取排行失败时会明确提示读取失败，不会把数据库异常误显示为暂无参与者。

## 修仙配置

- 修仙经济/成长参数写入仅限超级管理员私聊执行，必须填写原因并二次确认。
- 修仙配置写入会在同一数据库事务内完成配置更新、规则校验和审计记录；配置更新未命中会回滚并提示重试；事务提交后刷新配置缓存，若缓存刷新失败会明确提示需要管理员核查。
- 非凡人大境界突破配置保留生产保护：成功率必须大于 0% 且不超过 95%，积分消耗必须大于 0，冷却不得低于 1 小时，最低修为必须大于 0 小时。
- `突破` 检查自动代购前会读取钱包和突破丹库存；读取失败时提示稍后重试，不会误显示为 0 积分、余额不足或无丹药后继续代购。
- `修仙榜` 会展示全服修仙排行；已加入宗门的上榜道友会在用户名后显示所属宗门。

## 管理员调账

- `操控用户积分` 仅限超级管理员私聊执行，必须填写原因并二次确认。
- 超级管理员可对任意已存在本地档案的 Telegram 用户调账，包括普通管理员、其他超级管理员和自己。
- 管理员授权、白名单、调账、模拟过期、暂停/恢复和删除用户等高危入口读取目标档案失败时会中止并提示稍后重试，不会误当成查无此人继续或结束流程；设置白名单禁止作用于自己和超级管理员。
- `暂停/恢复` 调用 ABS 失败或 ABS 已更新但本地同步失败时都会写失败审计；若失败审计也写入失败，会通知超级管理员人工核查。
- 调账保留单次 `5000` 积分、每名超级管理员每日累计 `20000` 积分限额；额度暂时无法读取时会拒绝调账，并写入 `admin_adjust` 积分流水和 `ADJUST_POINTS` 审计。

## 管理员运维

- `使用续期卡` 若需要恢复已封禁账号，会在续期到账后调用 ABS 恢复；ABS 恢复失败或本地同步失败都会写失败审计，若失败审计也写入失败，会通知超级管理员人工核查。
- `备份数据` 仅限超级管理员私聊执行，必须填写原因并二次确认；加密备份发送成功或失败都会写审计，若审计写入失败会通知超级管理员人工核查。
- 自动备份调度读取最近成功日期、重试次数或最近尝试时间失败时会跳过本轮备份并记录错误，不会在状态不明时继续外发数据库备份。
- 自动备份尝试开始前写入最近尝试时间失败时会跳过本次备份；备份失败后的重试次数或最近错误写入失败时会通知超级管理员，避免重复外发或重试次数失真。
- 自动备份文件已发送但成功状态、消息 ID 或重试状态写入失败时，会通知超级管理员人工核查，避免备份已外发却因状态未落库而重复发送。

- 备份消息置顶已执行但本地置顶状态写入失败时，会通知超级管理员人工核查，避免备份群置顶和后台记录长期不一致。
- `备份状态` / `后台状态` 读取自动备份重试次数失败时会显示读取失败或状态暂不可用；`备份状态` 读取最近成功日期、时间、消息 ID、置顶状态或错误字段失败时也会明确显示异常，不会把状态错误伪装成 0 次重试、无记录或从未成功。
- 自动听书榜单生成时，如果部分 ABS 统计读取失败会记录后台诊断；如果本期所有用户统计都读取或解析失败，会跳过当期榜单发送，避免把外部统计故障误公告成无人上榜。
- `清理遗孀` 仅限超级管理员执行，必须填写原因并二次确认；ABS 服务端删除完成后会写 `CLEAN_WIDOWS` 审计，若审计写入失败会通知超级管理员人工核查。
- `后台状态` 为超级管理员只读运维入口，可查看生命周期巡检、生命周期最近错误、每日听书刷新、自动备份、消息队列、Telegram 发送队列、Telegram API 成功/失败、慢处理计数和最近 Telegram 错误。
- `后台状态` 读取生命周期、每日听书刷新或自动备份摘要配置失败时会显示读取失败或状态暂不可用，不会把状态错误伪装成 0、无错误、从未完成或从未成功。

- 每日听书刷新已执行但刷新时间、统计数量或最近错误状态写入失败时，会通知超级管理员人工核查，避免下一轮重复批量刷新 ABS。

- 生命周期巡检已执行但成功日期或最近错误状态写入失败时，会通知超级管理员人工核查，避免下一轮重复执行同一天巡检。

## 宗门任务

- `我的宗门`、宗门排行、宗门成员和宗门科技中的成员上限如果暂时读取失败，会显示“读取失败”，不会把宗门科技异常误显示为基础上限。
- `宗门任务` 页面里的每日奖励如果暂时读取失败，会显示“读取失败”，不会把宗门科技加成异常误显示为基础奖励。
- `宗门科技` 面板里的科技等级或效果暂时读取失败时会显示“读取失败”；升级确认和正式升级读取失败时会中止，不会按 0 级继续扣宗门资金或声望。
- 宗门周目标固定为：全宗签到 `100` 次、全宗净修为增长 `200` 小时、全宗完成个人任务 `60` 次。
- 每周一北京时间 `09:00` 后自动结算上一周；只有达成至少 1 项的宗门会发奖，并私信提醒宗主和长老。未达成不提醒；自动结算状态读取失败时会跳过本轮，不会在状态不明时继续发放宗门资金或声望。
- 宗门周目标自动结算已执行但已结算周状态写入失败时，会通知超级管理员人工核查，避免下一轮重复扫描同一周。
- `结算宗门周目标` 保留为手动补结算命令；每个宗门每周只能结算一次，只奖励宗门资金和宗门声望，不发个人奖励。
- 手动补结算会先确认上一周是否已结算；读取失败时会中止，不会把数据库异常当成未结算后继续发放宗门资金或声望。
- 基础奖励：达成 1 项为资金 `50` / 声望 `20`，达成 2 项为资金 `120` / 声望 `50`，达成 3 项为资金 `250` / 声望 `100`。
- 超额奖励按三项目标分别计算，单项最多计算 `200%` 超额；超额百分比合计 `x0.8` 折算额外资金，`x0.3` 折算额外声望，均向下取整。
- 领取个人每日任务奖励时，如果三项未全部完成，Bot 会列出尚未完成的具体项目和当前进度，方便补齐后再领取。
- 宗门等级升级由宗主或长老执行，消耗宗门资金和声望；升级成功后若成员上限暂时读取失败，回执会显示读取失败，不会误报为 0 人。

- 求书备注、补充信息、管理员备注和要求补充说明最多 300 字，可换行，但不得包含制表符或其他控制字符。
- 提交求书、补充信息、管理员接单、添加备注、要求补充或标记已上传/暂无资源时，工单状态、处理日志和审计会在同一数据库事务内提交；若日志或审计暂时写入失败，本次操作会提示失败并保留原状态，避免工单缺少追溯。
- 管理员标记“已上传”后，系统会从每个 ABS 媒体库最近 5 条入库记录中筛选近 20 分钟内最新的一本书，展示媒体库、书名、演播和封面预览；管理员确认后才会发布到大群。该群公告不置顶、不参与自动删除；封面读取失败或 Telegram 图片发送失败时会降级为纯文本公告。

## 群内小游戏

- 推牌九开放时间为每日 `18:00-19:55`，命令为 `发起牌九`，下注格式为 `押 1` 到 `押 5`。Bot 坐庄，每人两张牌，A=1，2-9 按牌面，10/J/Q/K=0，两张相加取个位；9 点最大，同点庄家大半点。每人每局只能下注一次，下注期 60 秒，局间冷却 1 分钟，最多 20 人；玩家输给庄家的下注会注入天道奖池；开局、状态和下注成功提示会展示更完整的规则/桌面信息，开奖结果参与群消息定时删除且不置顶。
- 赛马开放时间保持每日 `20:00-22:00`。
- 骰子开放时间调整为 `22:05-次日17:55`，避开推牌九和赛马时段。

## 线路

- `获取线路` 会展示管理员配置的服务器线路；线路正文会按 Telegram Markdown 安全转义，读取失败时会明确提示，不会误显示为未配置。
- `天道奖池` 查看当前全服机缘奖池；奖池配置暂时读取失败时会显示读取失败，不会误显示为 0。
- `财富榜` / `积分榜` 可在私聊或群聊查看普通用户积分 Top 50，每页最多 30 人；人数较多时通过消息底部按钮翻页，不支持文字页码翻页。
- 每日天道灵气自动注入会先读取当日收集状态；读取失败时跳过本轮并通知超级管理员，不会在状态不明时继续注入奖池。

## 卡密回收

在私聊打开 `资产交易 -> 卡密回收`，或发送 `卡密回收` / `回收卡密`。Bot 只接受本服生成且当前真实可用的邀请码、续期卡；输入完整卡密后会显示报价，回复 `确认回收卡密` 才会入账，回收成功后卡密立即失效且不可撤销。

回收价为当前配置定价的 60%：邀请码按当前邀请码价格计算；续期卡按当前 30 天续期价格和卡片天数折算后计算。默认定价下，邀请码回收 180 积分，3/30/90/365 天续期卡分别回收 9/90/270/1095 积分。实际以确认前 Bot 报价为准，若确认期间价格变化会中止并要求重新报价。

卡密采用持有码规则：只要卡密属实可用即可回收，不要求是原生成者或登记所有者。因此请妥善保管明文；曾接触过卡密的人可能抢先提交，但同一卡密只会成功回收一次。若卡密正在交易行出售，成功回收会同步关闭对应可售卡密，避免继续成交。
## 交易行

- `交易行 关键词` 可按商品名或背包物品名筛选，关键词为 1-40 个字。
- 筛选关键词不得包含换行、制表符或其他控制/分隔字符。
- `上架商品` 私聊分为自由卡密和背包上架：自由上架商品名为 2-40 个字，背包物品名为 1-40 个字，二者都不能包含换行、制表符、其他控制字符或 Unicode 行/段分隔符。
- 背包丹药商品会在交易行列表、详情、购买确认、购买成功、卖家成交通知和上架提醒中显示简要功效，方便买卖双方确认用途；自由卡密同名商品不显示系统丹药功效。
- 自由卡密仅允许上架 Bot 生成且尚未使用的邀请码/续期卡，成交前会再次复核仍未使用；积分兑换续期卡只能由当前持有人上架，成交后归属转给买家；复核失败会中止交易并暂停该商品待核查。
- 交易行商品单价必须保持在系统原价值的 `85%`-`115%` 范围内；邀请码/续期卡按当前系统价格折算，背包物品按系统可识别的商城价、种子价、药草基础价或炼丹成本价折算，无法识别原价值的特殊物品仍受全局价格上限限制但不套用原价值区间。
- 交易行库存读取失败时会明确显示读取失败或提示库存状态暂不可用，不会把查询错误当成无库存或可购买。
- 管理员查询交易订单时，争议记录读取失败会明确显示读取失败，不会把数据库异常误显示为无争议记录；超级管理员可私聊使用 `强制下架商品 商品ID`，输入原因并二次确认后强制下架在售商品。
- 交易行商品有效期为 48 小时；到期后 Bot 自动下架，背包商品未售库存会退回卖家乾坤袋；商品售罄后会自动结束在售状态，不会再发送到期下架提醒。
- 若商品卖家数据出现不一致，系统会暂停该商品交易并通知管理员核查，不会继续自动下架或错误退回库存。

## 宗门喇叭

- `宗门喇叭 内容`：宗主或长老私聊使用，2 积分一次，通知本宗门成员。
- `世界喇叭 内容`：宗主或长老私聊使用，5 积分一次，通知已绑定 ABS、未停用且未封禁的正式用户。

## 个人邀请拉新

- 达到炼气初期及以上的正式用户，私聊发送 `我的邀请`、`邀请链接` 或 `拉新链接` 可获得个人邀请链接。
- 新人通过邀请链接注册后获得 7 天新人体验权限。
- 邀请链接校验或注册归因遇到数据库读取异常时，会提示稍后重试，不会把系统故障误报为链接无效。
- 新人在体验期内累计听书满 10 小时后，私聊发送 `新人任务` 可领取 7 天新人体验延期；任务统计按试用开始到当前/结束的实际墙钟时长封顶，避免把试用开始前的当日历史听书误算入任务。
- 被邀请者不获得积分；邀请者每产生 1 名有效新人获得 10 积分，每月最多 150 积分。
- 同一邀请链接每天最多激活 3 名新人体验。
- 私聊发送 `邀请统计` 可查看累计激活、有效新人和本月奖励；若统计暂时读取失败，Bot 会明确显示读取失败，不会用 0 代替。
- 新人体验延期不是普通续期卡，不能转赠、囤积或给正式用户使用。
- 试用用户不能使用普通续期卡；需要先使用正式邀请码转正，转正后才能使用普通续期卡。
