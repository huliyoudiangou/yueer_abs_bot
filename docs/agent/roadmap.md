# Project Roadmap for Agent

维护流程与文档同步要求见 `docs/agent/maintenance.md`。

## 当前项目主线

悦耳声阅 Bot 的核心是：

1. ABS 用户生命周期管理；
2. 积分经济系统；
3. 听书行为游戏化；
4. 社群互动；
5. 管理员可控运维；
6. 生产安全与数据恢复。
   任何新功能都应服务这些主线，而不是变成无关工具集合。

## 可拓展方向

### 修仙系统

可以拓展：

- 灵根：影响修为转化、突破成功率、丹药收益。
- 功法：通过签到、Boss、宗门兑换获得，提供长期加成。
- 法宝：消耗积分或活动获得，提供突破保护、Boss 加成。
- 秘境：限时听书活动，按新增净修为结算奖励。
- 雷劫记录：更完整展示突破历史和失败补偿。
- 境界配置后台：更完整的增删改查，而不是只改部分字段。
  边界：
- 不能让付费丹药无限堆叠导致经济崩坏。
- 不能让突破失败造成不可恢复的资产损失。
- 必须保留配置校验和管理员审计。

### 宗门系统

可以拓展：

- 宗门仓库；
- 宗门公告；
- 宗门申请审批；
- 宗门战；
- 宗门拍卖；
- 宗门任务刷新；
- 长老权限细分；
- 宗门日志查询。
  边界：
- 宗门资金、贡献、声望必须有流水或日志。
- 宗主转让、踢人、职位变更必须记录审计或宗门日志。
- 不能允许普通成员消耗宗门公共资产。

### 经济系统

可以拓展：

- 拍卖行；
- 玩家间红包增强；
- 限时商店；
- 积分税；
- 积分排行榜历史；
- 异常积分告警；
- 管理员回滚工具。
  边界：
- 玩家间直接交易必须防诈骗、防并发、防重复发货。
- 所有积分变动必须可追溯。
- 管理员调账必须保留额度限制和原因记录。

### 运营系统

可以拓展：

- 节日活动；
- 自动公告；
- 活动配置表；
- 活动奖励池；
- 活跃用户召回；
- 到期用户分层提醒。
  边界：
- 自动任务必须幂等。
- 不要在群里刷屏。
- 所有定时任务要能重启后安全恢复。

### 管理系统

可以拓展：

- 用户资产全景查询；
- 积分异常检测；
- 修仙配置完整管理；
- 后台运维状态查询：数据库备份、生命周期巡检、ABS 同步/刷新任务的只读健康面板；
- ABS 同步状态巡检；
- 管理员操作报表。
  边界：
- 普通管理员只读或低风险。
- 超级管理员写操作需要确认。
- 敏感数据只展示脱敏版本。
