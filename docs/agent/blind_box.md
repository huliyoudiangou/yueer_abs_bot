# 积分盲盒维护约束

- 盲盒扣积分、邀请码或续期卡创建必须在同一个数据库事务内完成。
- 扣分失败、奖品创建失败或事务提交失败时，不得向调用方返回中奖卡密、邀请码或全服公告文案。
- 成功回执和大奖广播内容只能在事务成功后发布给外层发送逻辑；错误路径必须返回空消息和错误，避免出现“卡密未落库但已私聊发出”或“扣分回滚但公告已生成”。
- 每次开启固定消耗 25 积分。当前奖品概率固定为：谢谢惠顾 73.9%、3 天续期卡 20%、专属邀请码 2%、30 天续期月卡 3%、90 天续期卡 1%、365 天尊享年卡 0.1%。随机数使用 1-1000 的整数区间，对应边界依次为 739、939、959、989、999、1000。调整价格或概率属于生产经济参数变更，必须同步测试和经济文档。
- `blind_box_logic_test.go` 守护 `executeBlindBoxOpen` 的事务返回模式，禁止回退为直接返回事务内部写入的消息变量。
